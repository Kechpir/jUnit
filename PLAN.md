# План выполнения тестового задания по автоматизации (Java + Selenium)

Этот документ описывает шаги по реализации автоматизированных тестов для сайта [SauceDemo](https://www.saucedemo.com/).

## 1. Инициализация проекта
- [ ] Создание структуры проекта (Maven или Gradle).
- [ ] Настройка `pom.xml` / `build.gradle` с необходимыми зависимостями:
    - Selenium WebDriver
    - JUnit 5
    - Allure Reports
    - WebDriverManager (для управления драйверами)
    - Log4j2 / SLF4J (для логирования)

## 2. Разработка инфраструктуры
- [ ] Создание базового класса для тестов (`BaseTest`) с логикой запуска/закрытия браузера.
- [ ] Реализация поддержки нескольких браузеров (Chrome, Firefox).
- [ ] Настройка конфигурационных файлов (URL, таймауты).
- [ ] Настройка логирования действий в тестах.

## 3. Реализация паттерна Page Object Model (POM)
- [ ] Создание `LoginPage`:
    - Локаторы для полей ввода и кнопки входа.
    - Методы для ввода данных и нажатия кнопки.
    - Методы для получения сообщений об ошибках.
- [ ] Создание `ProductsPage`:
    - Локаторы для проверки успешного входа (например, заголовок "Products").

## 4. Написание тестовых сценариев
- [ ] **Test 1**: Успешный логин (`standard_user`).
- [ ] **Test 2**: Логин с неверным паролем.
- [ ] **Test 3**: Логин заблокированного пользователя (`locked_out_user`).
- [ ] **Test 4**: Логин с пустыми полями.
- [ ] **Test 5**: Логин `performance_glitch_user` (проверка загрузки страницы с ожиданием).

## 5. Отчетность и документация
- [ ] Интеграция Allure (скриншоты при падении, шаги `@Step`).
- [ ] Создание `README.md` с инструкциями по запуску.
- [ ] Подготовка Git-репозитория:
    - Ветки `master` и `dev`.
    - Оформление Pull Request.

## 6. Финализация
- [ ] Проверка чистоты кода и именований.
- [ ] Финальный прогон всех тестов.
