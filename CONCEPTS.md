# Справочник концепций QA Automation (Java)

## 1. Maven (pom.xml)
Это "паспорт" проекта. Здесь мы указываем, какие библиотеки нам нужны. Maven сам скачивает их из центрального репозитория.

## 2. WebDriver & WebDriverManager
- **WebDriver**: Интерфейс для управления браузером.
- **WebDriverManager**: Библиотека, которая избавляет от мучений с поиском и скачиванием `chromedriver.exe`. Она делает всё сама.

## 3. JUnit 5 Annotations
- `@Test`: Помечает метод как тест.
- `@BeforeEach`: Выполняется ПЕРЕД каждым тестом (удобно для настройки драйвера).
- `@AfterEach`: Выполняется ПОСЛЕ каждого теста (удобно для закрытия браузера).
- `@DisplayName`: Красивое имя теста для отчетов.

## 4. Page Object Model (POM)
Разделение кода на:
1. **Pages**: Описание элементов и действий (где кнопка и как на неё нажать).
2. **Tests**: Сами проверки (ввели логин, нажали кнопку, проверили результат).

## 5. Allure
Система отчетности. Позволяет видеть шаги теста, скриншоты ошибок и время выполнения в виде веб-сайта.
